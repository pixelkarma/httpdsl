⚡ httpdsl vs Native Go — Benchmark

wrk: 4 threads, 50 connections, 10 seconds per test

Testhttpdsl (req/s)Go (req/s)RatioVerdict

Analysis: For typical API work (route matching, JSON in/out, string ops), httpdsl achieves 85-90% of native Go throughput — the bottleneck is network I/O and JSON serialization, not the interpreter.

For computation-heavy routes (tight loops, recursive calls), the tree-walking interpreter pays 7-14x overhead from AST node allocation, scope creation, interface boxing, and mutex acquisition per variable access.

Bottom line: If your routes do what APIs typically do (validate input, query a store, return JSON), httpdsl is fast enough for production. If you need computation, do it in Go and call it as a builtin.
