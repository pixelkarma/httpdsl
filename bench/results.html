Optimization journey (3 rounds):
  1. Tree-walker baseline — map-based scopes, mutex on every access, AST recursion
  2. Interpreter optimizations — skip mutex for per-request envs, inline append, singleton signals
  3. Bytecode VM — compile AST to bytecodes, stack-based VM with fixed-size stack array, slot-indexed locals (no map lookups), int64 fast paths for arithmetic/comparison, dedicated OP_INC_LOCAL and OP_APPEND opcodes, zero-alloc dispatch loop
Final results: API workloads remain at 84-88% of Go (I/O bound, JSON serialization dominates). Fibonacci computation: 9,780 → 34,812 rps (3.6x faster), now at 49% of Go. Array building: 5,033 → 38,692 rps (7.7x faster), now at 54% of Go.

Remaining gap: ~2x on computation comes from Go's interface{} boxing — every Value goes through an interface, which the Go compiler can't optimize away. A JIT compiler emitting native code is the only way past this wall.
